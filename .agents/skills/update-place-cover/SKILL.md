# Update Place Cover

## Overview

Обновляй обложку, миниатюру маркера и первый слайд фотокарусели существующего места единым helper-скриптом. Не конвертируй изображения вручную и не редактируй несвязанные поля места.

## Required Inputs

- Локальный путь к изображению.
- `placeId` или точное название места.
- Название карты — только если совпадение без него неоднозначно или пользователь явно указал карту.

## Workflow

1. Прочитай `.specify/memory/constitution.md` и проверь незакоммиченные изменения.
2. Убедись, что исходное изображение существует локально. Не ищи и не скачивай другое изображение без запроса пользователя.
3. Запусти helper с `--dry-run` и проверь найденное место, файл данных, новые WebP-пути, номер слайда и список старых ассетов для удаления.
4. Если место не найдено или совпадение неоднозначно, уточни ID или карту. Не выбирай совпадение наугад.
5. Если dry-run соответствует запросу, повтори команду без `--dry-run`.
6. Проверь JSON, формат и размеры созданных файлов, diff и уместные существующие проверки проекта. Новые тесты добавляй только с явного разрешения пользователя.
7. Добавь или обнови выполненную задачу текущей фичи в `specs/<feature>/tasks.md`; `spec.md` и `plan.md` меняй только при изменении требований или подхода.

## Commands

По ID:

```bash
node .agents/skills/update-place-cover/scripts/update-place-cover.mjs \
  --image "/absolute/path/photo.heic" \
  --place-id 1410 \
  --dry-run
```

По названию и карте:

```bash
node .agents/skills/update-place-cover/scripts/update-place-cover.mjs \
  --image "/absolute/path/photo.jpg" \
  --place-name "Лавандовый берег" \
  --map-title "Куда в Курске"
```

## Guarantees

- Обложка получает длинную сторону не более 1600 px; миниатюра — центральный квадрат 480×480.
- Оба файла создаются в WebP с нормализованной ориентацией и hash-именами.
- При наличии `balloonContent.images` helper синхронизирует первый отображаемый слайд, сохраняя `order` и `caption`.
- Старые локальные ассеты удаляются только когда больше не используются ни в одном `public/data/*.json`.
- Не создавай ветку или worktree и не затрагивай чужие незакоммиченные изменения.

## Common Mistakes

- Не запускай `sips`, `qlmanage` или `cwebp` вручную вместо helper-а.
- Не перезаписывай стабильный URL старого ассета: generator создаёт hash-имя для обхода кэша.
- Не удаляй старый файл по одному `rg`; helper проверяет ссылки во всех JSON-картах.
