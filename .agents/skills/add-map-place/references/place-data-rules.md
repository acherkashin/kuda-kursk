# Правила данных места

## Формат GeoJSON

Новое место добавляй как объект `Feature` в `features[]` нужного файла из `public/data/`.

Минимальный объект:

```json
{
  "type": "Feature",
  "id": 123,
  "geometry": {
    "type": "Point",
    "coordinates": [36.193015, 51.730846]
  },
  "properties": {
    "id": 123,
    "balloonContent": {
      "image": "/place-images/123-image-name.webp",
      "thumbnail": "/place-thumbnails/123-thumbnail-name.webp",
      "name": "Название места",
      "description": "Описание места.",
      "address": "Курск, улица Ленина, 1"
    }
  }
}
```

## Координаты

- `geometry.coordinates`: строго `[longitude, latitude]`.
- `balloonContent.coordinates` не записывай: строка `"latitude, longitude"` вычисляется из `geometry.coordinates`.
- Для Курска и Курской области ожидаемые диапазоны: longitude `35..38`, latitude `50..53`.
- Если координаты похожи на `[latitude, longitude]`, не записывай их.

## Карты

- Главная карта по умолчанию: `Куда в Курске`, файл `public/data/main-map.json`.
- Остальные карты ищи в `src/domain/mapCatalog.ts` по `title` или `slug`.
- Если пользователь указал название карты не точно, выбери только однозначное совпадение. При неоднозначности спроси.

## Интернет-источники

- Ищи в интернете только недостающие поля после согласия пользователя.
- Не добавляй неподтверждённые режимы работы, цены, статусы, события, награды и обещания доступности.
- Для найденных адреса, названия или описания покажи пользователю, из каких источников взяты факты.
- Для найденной картинки покажи URL и источник; не скачивай и не записывай её без явного подтверждения пользователя.
