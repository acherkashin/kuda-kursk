# Контракт маршрутов SPA

## `/`

Основная публичная карта Курска.

**Поведение**
- Показывает все публичные места.
- Центрирует карту на Курске.
- Поддерживает поиск, карточку места, route actions и consent UI.

## `/maps/:slug`

Карта сообщества по прямой ссылке.

**Параметры**
- `slug`: URL-safe идентификатор подготовленной карты.
- `place` query-param: необязательный стабильный идентификатор места для открытия карточки по прямой ссылке.
- `zoom` query-param: необязательный масштаб карты MapLibre.
- `category` query-param: необязательный slug единственной выбранной категории мест.

**Поведение для известного slug**
- Показывает название и контекст карты сообщества.
- Показывает публичные места из `placeIds` и link-only места из `linkOnlyPlaceIds`.
- Карточка места работает так же, как на основной карте.
- Если URL содержит `/maps/:slug?place=:placeId` и место есть в наборе текущей карты, приложение автоматически выбирает место, фокусирует карту на его координатах и открывает карточку.
- Если `place` не совпадает с местом текущей карты, приложение закрывает карточку и очищает только невалидный `place` из URL.
- Отправляет `community_map_opened` после analytics consent.

**Поведение для неизвестного slug**
- Показывает понятное fallback-состояние.
- Даёт действие возврата на `/`.
- Не представляет карту как закрытое пространство или ошибку прав доступа.

## URL state

Выбранное место фиксируется в query-param `place`: `/maps/<slug>?place=<id>`. Значение `place` равно публичному `PlaceFeature.id`, нормализованному как строка. Закрытие карточки удаляет только `place`, сохраняя путь текущей карты и другие query-параметры.

Масштаб карты фиксируется в query-param `zoom`: `/maps/<slug>?zoom=12.5`. При открытии страницы валидное числовое значение в диапазоне MapLibre `minZoom`/`maxZoom` применяется к карте; если `zoom` отсутствует или невалиден, используется дефолтный масштаб карты. Изменение масштаба карты обновляет только `zoom` через replace-навигацию, сохраняя `place`, `about` и другие query-параметры без добавления каждого изменения масштаба в историю браузера.

Категория фиксируется в query-param `category`: `/maps/<slug>?category=chalet`. Допустим только slug из каталога категорий, представленный хотя бы одним местом текущей карты. Выбор нового chip заменяет предыдущее значение, повторный выбор удаляет `category`, а неизвестный или недоступный slug очищается через replace-навигацию. Категория сохраняет совместимые query-параметры; если `place` указывает на место вне выбранной категории, приложение закрывает карточку и удаляет только `place`. Ручной выбор категории имеет приоритет над текущим `zoom` и вписывает все её места в видимую область; при первоначальном открытии явные `zoom` или `place` имеют приоритет, а URL только с `category` запускает автоматический fit. Завершённый fit записывает новый `zoom` через общий контракт карты. Снятие категории и изменение поиска сохраняют текущую камеру.

Поиск остаётся локальным состоянием приложения и не фиксируется в URL.
