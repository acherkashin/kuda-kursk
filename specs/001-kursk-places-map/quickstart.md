# Быстрый старт: Куда в Курске

## Подготовка

1. Установить зависимости frontend-проекта:

```bash
pnpm install
```

2. Для локальной проверки Метрики задать ID счётчика только в `.env.local`:

```bash
VITE_YANDEX_METRIKA_ID=00000000
```

Если переменная не задана, analytics adapter должен работать в no-op режиме.

## Локальный запуск

```bash
pnpm dev
```

Ожидаемо: открывается SPA с полноэкранной картой Курска, маркерами, поиском и фильтрами. До согласия пользователя script Яндекс.Метрики не загружается.

## Проверки качества

```bash
pnpm typecheck
pnpm test:unit
pnpm test:e2e
pnpm build
```

Ожидаемо: TypeScript проходит без ошибок; unit-тесты покрывают фильтры, поиск, координаты и route links; Playwright проходит desktop/mobile/PWA/analytics сценарии; production build создаёт PWA manifest и service worker.

## Проверка PWA

1. Собрать production build:

```bash
pnpm build
pnpm preview
```

2. Открыть preview в Chromium.
3. Проверить manifest, icons 192/512, service worker registration и offline app shell.
4. Убедиться, что внешние tiles OpenFreeMap не кешируются без ограничений и что приложение корректно показывает резервное состояние при недоступной карте.

## Проверка аналитики

1. Открыть приложение в новом профиле браузера.
2. До выбора в cookie/analytics banner убедиться, что:
   - `window.ym` отсутствует;
   - запрос к script Метрики не выполнялся;
   - analytics cookies/localStorage Метрики не появились.
3. Нажать "Принять аналитику".
4. Убедиться, что script Метрики загружен и события проходят через typed adapter.
5. Выполнить поиск и проверить, что событие `search_used` не содержит сырой текст запроса.
6. Отозвать согласие через настройки аналитики и убедиться, что будущие события не отправляются.

## Проверка карт сообществ

1. Открыть `/community/<known-slug>` и убедиться, что отображается отдельная идентичность карты и нужный набор мест.
2. Открыть `/community/unknown-slug` и убедиться, что приложение показывает понятное fallback-состояние и путь назад на основную карту.
